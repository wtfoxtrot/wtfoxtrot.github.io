Misconfigured CSRF

There are several places too look. Most likely a type

 

1) On the app.js, check the csrfProtection for typos: const csrfProtection = csrf({ cookie: true }); This got me deferred >:( 2) Make sure csrfProtection is being passed as a parameter (see#2 in pic). 3) Make sure the token(invoked) is passed in the response object (see #3 in pic). 4) Make sure the input value in the Pug file corresponds to the key value in the response object.

Cannot read property of 'method' of undefined.

Ya done spelled something wrong.

 

1) Make sure the imported names match the table column name. Ex: {Pasta, Noodle, Sauce} = require('./models');

Insert or update on table "TabeName" violates foreign key constraint "Table_columnNameId_fkey"

One table is referencing a non-existent ID from another table. This could happen if you un-seed and then re-seed, but the primary keys did not reset OR if the foreign key is simply not valid.

 

1) Drop table and then re-migrate and re-seed. 2) In PostBird, manually change the primary keys to match the appropriate references. 3) In the seeder file, make sure the foreign key references something that exists.

ModelName.belongsTo called with something that's not a subclass of Sequelize.Model

1) Check the model the .belongsTo is referencing. Most likely a typo. Remember Models should be singular

Browser hangs after configuring CSRF protection

After trying to configure CSRF protection, going to any route fails, and the website appears to be constantly loading. If the server is configured with morgan, GET requests show blank data in the terminal with the running server.

 

Check if the syntax for setting up the cookieParser middleware. The correct syntax is app.use(cookieParser()); If cookieParser is not invoked [e.g. app.use(cookieParser)], the observed problem appears.

Error: Could not connect to server: No such file or directory

This problem may appear when trying to open PostgreSQL via the command psql.

 

Solution #1 (For Windows):
In terminal, enter sudo service postgresql restart. This may need to be done each time the computer is rebooted.
 Solution #2 (For Mac):
1) In terminal, enter brew services restart postgres.
1) In terminal, enter rm /usr/local/var/postgres/postmaster.pid.

TypeError: [something].[method] is not a function

The [something] does not have the [method].

 

  • Check the given line number for typos. For example, the method findAll is not findALL or findall.
  • If [something] is a Model in Sequelize, check that this Model is imported in using require.
  • If the given line looks OK, check if the function surrounding that line contains typos. In one case, the problem was that the asyncHandler helper function had typos, with (req, req, next) being used as parameters, instead of (req, res, next). This was on a completely different line that the line mentioned in the error.
